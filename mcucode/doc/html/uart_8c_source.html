00064 /*
00065  * Receive a character from the UART Rx.
00066  *
00067  * This features a simple line-editor that allows to delete and
00068  * re-edit the characters entered, until either CR or NL is entered.
00069  * Printable characters entered will be echoed using uart_putchar().
00070  *
00071  * Editing characters:
00072  *
00073  * . \b (BS) or \177 (DEL) delete the previous character
00074  * . ^u kills the entire input buffer
00075  * . ^w deletes the previous word
00076  * . ^r sends a CR, and then reprints the buffer
00077  * . \t will be replaced by a single space
00078  *
00079  * All other control characters will be ignored.
00080  *
00081  * The internal line buffer is RX_BUFSIZE (80) characters long, which
00082  * includes the terminating \n (but no terminating \0).  If the buffer
00083  * is full (i. e., at RX_BUFSIZE-1 characters in order to keep space for
00084  * the trailing \n), any further input attempts will send a \a to
00085  * uart_putchar() (BEL character), although line editing is still
00086  * allowed.
00087  *
00088  * Input errors while talking to the UART will cause an immediate
00089  * return of -1 (error indication).  Notably, this will be caused by a
00090  * framing error (e. g. serial line "break" condition), by an input
00091  * overrun, and by a parity error (if parity was enabled and automatic
00092  * parity recognition is supported by hardware).
00093  *
00094  * Successive calls to uart_getchar() will be satisfied from the
00095  * internal buffer until that buffer is emptied again.
00096  */
